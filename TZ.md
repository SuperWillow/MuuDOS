# ТЗ — DOSDisk (DOS в браузере с диском из облака)

## Цель
Браузерный клиент DOS-эмулятора (js-dos v8), который использует внешнее хранилище
(в первую очередь GitHub-репозиторий) как «диск C:»: грузит файлы из хранилища в эмулятор,
даёт работать в DOS, и (фаза 2) сохраняет изменения обратно.

## Вход
- Источник диска:
  - GitHub: `owner/repo`, ветка, путь (опц.), токен (опц. — для приватных / для записи).


## Выход
- Запущенный DOS с примонтированным диском C: из выбранного источника.
- (Фаза 2) Изменённые файлы выгружены обратно: скачать как ZIP и/или push в GitHub.

## Критерии готовности (DoD по фазам)
- **Фаза 1:** с публичного GitHub-репо и по прямой ссылке диск монтируется и DOS стартует;
  ручной чек-лист пройден, отчёт стажёра есть.
- **Фаза 2:** после изменения файлов в DOS их можно скачать ZIP-ом; push в GitHub работает на тестовом репо.
- **Фаза 3:** только после согласования с ПМ (OAuth-секреты).

## Ограничения / решения
- **Движок:** js-dos v8 (а не классический dosbox.js — тот только чтение).
- **Безопасность:** GitHub-токен и любые OAuth-секреты вводит пользователь в браузере;
  мы их не хардкодим и не коммитим. 
- **Хостинг:** статика (GitHub Pages). Часть ассетов js-dos грузится с CDN.
- **CORS:** прямые ссылки/чужие API ограничены CORS; GitHub raw/API — дружелюбны.

## Открытые вопросы

1. Нужна ли запись в приватные репо (тогда обязателен токен с правом `repo`)?
2. Лимиты на размер диска (большой репо = долгая загрузка в память эмулятора)?
