# programs/ — каталог программ для команды `INSTALL`

DOS-команда **`INSTALL`** в GitDOS ставит программы отсюда (из репозитория проекта `sementsul/GitDOS`).

## Раскладка
```
programs/
  <категория>/
    <id>/
      install.json     # манифест (обязателен)
      ... файлы программы ...
```
Категории — любые папки: `file-managers`, `games`, `dev`, `antivirus`, `tools`, …

## install.json
```json
{
  "name": "Norton Commander",   // отображаемое имя
  "id": "nc",                   // короткий id для команды INSTALL nc
  "dir": "NC",                  // куда ставить: C:\PROGRAMS\NC
  "run": "NC.EXE",              // (опц.) чем запускать
  "path": true,                 // (опц., по умолч. true) добавить C:\PROGRAMS\NC в PATH
  "version": "5.0",             // (опц.) версия — в списке и INFO
  "desc": "Двухпанельный...",   // (опц.) описание — в списке и INFO
  "autoexec": [],               // (опц.) доп. строки в AUTOEXEC.BAT
  "config": []                  // (опц.) доп. строки в CONFIG.SYS (драйверы)
}
```

## Где лежит каталог
По умолчанию INSTALL читает из `sementsul/GitDOS` (папка `programs/`). Источник можно сменить в GitDOS:
«⚙️ Общие настройки» → «🧩 Каталог программ» — указать `owner/repo` или `owner/repo/подпапка`
(например, отдельный репозиторий-каталог `sementsul/gitdos-programs`).

## Команда в DOS
- `INSTALL` — список доступных программ (по категориям), `*` = уже установлена.
- `INSTALL <id>` — скачать файлы программы в `C:\PROGRAMS\<dir>` и прописать в конфиги
  (добавить в PATH, плюс строки `autoexec`/`config`). Управляемый блок помечается `REM GITDOS:<id>`.

## Безопасность
Манифест может добавлять строки в AUTOEXEC.BAT/CONFIG.SYS — это исполняемый код. Поэтому источник
фиксирован на доверенный репозиторий проекта (`sementsul/GitDOS`). Кладите сюда только проверенный софт
(у DOS-программ — свои лицензии; добавляет владелец репозитория).
