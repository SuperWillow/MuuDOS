📖 База знаний GitDOS

Что такое GitDOS

DOSBox (DOS) прямо в браузере, где диск хранится в репозитории GitHub. Движок — js-dos v8, лежит локально в проекте (vendor/js-dos/). Сетевые обращения: api.github.com (сам сервис, не сторонний CDN) и аналитика — Яндекс.Метрика (mc.yandex.ru, 110203712, Вебвизор) и Google Analytics (G-QSFQDH69QV).

Диски и репозитории

  • Диск C: — основной репозиторий (его файлы лежат в корне C:).
  • Дополнительные репозитории — папки внутри C:\DISKS\ (например C:\DISKS\D). js-dos монтирует только один диск, поэтому доп. диски — это папки на C:, каждая со своей синхронизацией.
  • Подключение: окно «💾 Диски и репозитории». Вкладка Существующий (owner/repo, ветка, путь, токен) или Создать новый (имя + приватный).

Токен и сохранение в пароли

Для приватных репозиториев и записи нужен токен GitHub с правом repo. Токен уходит только на api.github.com, в код/репозиторий не попадает. По кнопке «Подключить и сохранить» браузер предлагает сохранить данные в менеджер паролей и подставляет их при следующем заходе.

Запуск

  • Кнопка ▶ Пуск монтирует диск и стартует DOSBox (без стартового окна js-dos).
  • При запуске на диске проверяются AUTOEXEC.BAT и CONFIG.SYS — если их нет, создаются по шаблону; существующий AUTOEXEC.BAT выполняется.
  • ⛶ Полный экран — развернуть экран DOSBox.

Загрузка файлов с компьютера

Окно «📤 Загрузка файлов» (или перетаскивание): файл попадает в запущенный эмулятор (диск C:) и в его репозиторий.

Автосохранение на GitHub

  • Работает само: раз в ~6 секунд проверяет изменения диска.
  • Пушит только новые/изменённые файлы (сравнение по git-sha), удаляет в репозитории то, что удалили в DOS.
  • Каждый диск пишет в свой репозиторий. Для записи нужен токен; без него — только чтение.
  • Финальное сохранение — при «■ Стоп».

Как устроен мультидиск (технически)

js-dos в этой сборке: (1) монтирует только один диск C:; (2) его загрузчик образа не создаёт подпапки. Поэтому в образ кладутся только файлы корня C:, а подпапки (диск D: и вложенные каталоги) дописываются после старта через fsWriteFile, после чего диск пересобирается через persist и C: перемонтируется — так все папки видны с самого старта.

Язык интерфейса

Интерфейс на русском и английском. По умолчанию выбирается язык браузера; переключатель RU/EN — в правом верхнем углу (выбор запоминается).

Установка как приложение (PWA)

GitDOS — прогрессивное веб-приложение: его можно установить на телефон/десктоп (значок «Установить» в адресной строке браузера) и запускать как обычную программу. Оболочка кэшируется сервис-воркером, поэтому интерфейс работает и офлайн (для синхронизации с GitHub нужна сеть).

Приватность и движок

Все библиотеки локальные. Токены — только в браузере (localStorage/менеджер паролей). Дизайн — в стиле установщика Windows 98. На старте DOS-экрана выводится ASCII-логотип GitDOS (файл GITDOS.NFO, печатается через TYPE; в репозиторий не синхронизируется).

ℹ️ О программе

GitDOS — DOSBox в браузере, где диски DOS хранятся в репозиториях GitHub.

Диск C: — основной репозиторий. Дополнительные репозитории подключаются как папки в C:\DISKS\ (C:\DISKS\D, C:\DISKS\E…) — js-dos монтирует только один диск. Каждая папка синхронизируется со своим репозиторием.
